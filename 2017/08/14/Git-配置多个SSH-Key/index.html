Git 配置多个SSH Key

发表于 2017-08-14

今天遇到了这种情况：自己的兴趣项目同时在github和gitoschina上，这样就需要在一台机器上配置多个SSH key对应到上述两个不同的环境。

下面是具体的操作步骤：

1. 生成github的ssh key

$ ssh-keygen -t rsa -C "luojunwu1988@163.com” -f ~/.ssh/github
$ ls ~/.ssh

然后将看到公钥文件：github.pub，将文件里面的内容拷贝出来，粘贴到github后台的SSH Key配置中

2. 生成gitoschina的ssh key

$ ssh-keygen -t rsa -C "luojunwu1988@163.com” -f ~/.ssh/gitoschina
$ ls ~/.ssh

然后将看到公钥文件：gitoschina.pub，同样将文件里面的内容拷贝出来粘贴到gitoschina后台的SSH Key配置中

3. 修改SSH的配置文件（最关键的一步）

在~/.ssh/目录下新建config文件

$ vim ~/.ssh/config

添加以下内容

Host github.com
    IdentityFile ~/.ssh/github
    User git

Host git.oschina.net
   IdentityFile ~/.ssh/gitoschina
   User git

4. 测试

$ ssh -T git@github.com
$ ssh -T git.oschina.net

如果配置正确，控制台都将打印欢迎语以及自己的用户名
